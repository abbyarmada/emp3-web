Extensible Map Platform (EMP) is a framework for displaying and manipulating geospatial data with Android and Web applications. This development guide provides an overview of the capabilities of EMP and how to use EMP within your application

EMP3 Web

The EMP3 Web capability is based from the first EMP iteration so has a comparable amount of capability as EMP with the addition of some new capabilities and refreshed user interface in the works.

Use Cases:
  • Integration into the COE v3 Web Client Plugin Framework
  • Integration into web applications as a native component
  • Use as a "widget" in Ozone Widget Framework and Ozone Platform
  • Use as a "widget" via HTML 5 standards such as HTML Messaging

EMP3 Android

EMP3 Android is brand new and in an early stage. The goal with the Android version is to implement a consistent architecture with the web version, a developer facing API that is consistent in naming, style, and capability set. Initially it will support the Android NASA WorldWind SDK.

  • Integration into the COE v3 Smart Client Plugin Framework
  • Integration into Android Applications API 19 (4.4.2) and newer

Architecture

EMP3 is broken into a modular design with 4 primary component areas:

  1. Map API
    • Acts as single, consistent interface for any application that uses the map framework
    • Currently Implements CMAPI 1.3.0 and CMAPI geospatial notation for joint interoperability
  2. Map Core
    • Performs majority of map internal business logic
    • Stores data hierarchy and manages instance counts within data hierarchies
    • Interacts with map engine through engine interface
  3. Map Engine Interface
    • Ensures consistent interface between map core and map engine implementation
    • Protects map API and Map Core from different internal interfaces used by map engine SDKs
    • Enables changing of map engine with least possible code overturn
    • Provides coverage of what a map engine implementation glue code must provide
  4. Map Engine Implementation
    • Glue Code
      • Translates data and events between 3rd party SDK and the normalized map engine interface types
      • Allows for integration of multiple 3rd party map SDKs without affecting development community using the EMP3 API
      • Establish clear contract for integration of new map SDKs as needed
    • 3rd Party Map SDK
      • Provides actual geospatial visualization of map, terrain, and feature data
      • Examples Are Cesium, Leaflet, NASA WorldWind
      • Developer interacting with EMP3 API has no knowledge of the specific SDK and can be confident their code will run consistently regardless
